Shopify App
M1 — Analytics Dashboard
Wire the following to live data from Alan's API:
•	Wire Total Value Protected — retail value of all enrolled shipments, with period comparison vs last period

•	Wire Engagement Funnel — Enrolled → Active → Expired with drop-off percentages and overall engagement rate

•	CORRECTION: Funnel state label currently reads "Verified" — change to "Active" throughout. "Verified" is not a proof lifecycle state.

M2 — Delivered State: Automatic Push
Priority: blocks proof lifecycle. Required before billing is accurate.
•	Subscribe to Shopify orders/fulfilled webhook
•	On fulfillment: match order to proof, call PATCH /api/proofs/:proof_id/delivered on Alan's backend automatically
•	Pass Shopify fulfillment timestamp as delivered_at — no merchant action required
•	Proof transitions to ACTIVE after COOLDOWN expires

M3 — Billing: Usage & Billing
Billing model: $2.99 per enrollment + $2.99 per verification (Active). All charges via Shopify Billing API only — never Stripe.
•	Wire current billing cycle — date range, Enrolled line item with count and amount, Active line item with count and amount, cycle total
•	Wire Usage Cap — display cap amount, amount used, percentage used, progress bar
•	Wire Usage History — per-order list showing order number, Enrolled or Active badge, timestamp, and charge amount
•	CORRECTION: Usage History badge currently shows "Verified" — change to "Active"
•	Export CSV — usage history export
•	When cap is reached: surface cap-reached state in dashboard, halt enrollment prompts

M4 — Settings: Inventory
•	Wire sticker inventory count with low-inventory indicator
•	Wire minimum order value control — stored as min_enrollment_value on merchant record via Alan's API
•	Orders below min_enrollment_value threshold are not enrolled — no proof created, no charge
•	Display reorder prompt when inventory falls below threshold — links externally to shop.in.ink
•	After confirmed purchase: call POST /admin/merchants/:shop_id/inventory/purchase with quantity and payment reference

M5 — Settings: Media

M6 — Settings: Notifications
All customer notifications sent via Twilio. Taimoor owns the Twilio integration for both email and SMS.
•	Integrate Twilio — configure credentials, sending domain, and SMS sender number
•	Wire Notification Channel toggles — Email and SMS (SMS requires customer phone number from Shopify order)
•	Wire Delivery Notifications — Out for delivery, Delivered (includes tap instructions), Delivery confirmed (sent after customer taps)
•	Wire Tap Reminders — 4 hours after delivery (first), 24 hours (second), 48 hours (final). Reminders stop once customer taps.
•	Wire Return Window Reminders — 7 days before window closes (includes return link), 48 hours before window closes
